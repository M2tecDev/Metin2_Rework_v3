# Metin2 Rework v3 — Wiki Generation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate a comprehensive GitHub Wiki (~50 pages) for all three submodule repos (client-src, client-bin, server-src), documenting every class, function, packet, and cross-cutting system flow.

**Architecture:** Four parallel agents generate repo reference pages in Phase 1. Nine parallel agents generate thematic cross-reference pages in Phase 2. One final agent generates navigation (Home.md + _Sidebar.md) in Phase 3. All output lands in `wiki/` at the repo root.

**Tech Stack:** Claude subagents (Explore + general-purpose), Markdown, GitHub Wiki (`wiki.git` repo), Bash for git operations.

**Spec:** `docs/superpowers/specs/2026-03-18-metin2-wiki-design.md`

**Repo root:** `D:/Git Repo/Metin2_Rework_v3/`

## Content Format Reference

All agents must follow these templates exactly.

### Reference Page Template (C++ libraries and Python modules)

```markdown
# <Module Name>

> <One-line purpose statement>

## Overview
<2-3 sentences: what this module does, why it exists, how it fits>

## Dependencies
- <OtherModule> — <why>

## Files
| File | Purpose |
|------|---------|
| `path/to/file.cpp` | <description> |

## Classes / Functions

### <ClassName>
**File:** `path/to/file.h`
**Purpose:** <what it represents>

#### Member Variables
| Variable | Type | Description |
|----------|------|-------------|
| `m_name` | `int` | <what it stores> |

#### Methods
| Method | Parameters | Returns | Description |
|--------|-----------|---------|-------------|
| `Foo(x)` | `int x` — the value | `bool` | <what it does> |
```

### Topic Page Template (cross-cutting flows)

```markdown
# <Topic Name>

## Overview
<What this flow/system does and why it matters>

## Flow
1. **Python** `file.py::FunctionName()` — <what happens>
2. **Packet** `CG_NAME (0xXXXX)` — <what it carries>
3. **Server** `file.cpp::HandlerName()` — <what happens>
...

## Key Files
| File | Repo | Role |
|------|------|------|
| `path/to/file` | client-bin | <role> |

## Packet Reference
| Packet | Direction | Payload | Trigger |
|--------|-----------|---------|---------|
| `CG_LOGIN3 (0x0102)` | Client → Game | username, password | Login click |
```

### Page-Size Rule

If a library/directory has more than ~20 source files, split into sub-pages. The parent page is an index.
- GameLib: `client-src-GameLib.md` (index) + `client-src-GameLib-Characters.md`, `client-src-GameLib-Combat.md`, `client-src-GameLib-Items.md`, `client-src-GameLib-Skills.md`

- server game/ (268 files): `server-src-game.md` (index) + `server-src-game-Character.md`, `server-src-game-Combat.md`, `server-src-game-Item.md`, `server-src-game-Quest.md`, `server-src-game-Network.md`, `server-src-game-Skill.md`, `server-src-game-Guild.md`
